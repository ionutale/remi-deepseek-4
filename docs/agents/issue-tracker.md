# Issue tracker: Local Markdown

Issues and PRDs for this repo live in a single `issues.md` file at the repo root.

## Conventions

- Issues are appended to `issues.md` as they are created
- Each issue is prefixed with a `Status:` line (see `triage-labels.md` for the role strings)
- PRDs are also recorded in `issues.md`

## When a skill says "publish to the issue tracker"

Append a new entry to `issues.md` at the repo root.

## When a skill says "fetch the relevant ticket"

Read `issues.md` and find the relevant entry.
